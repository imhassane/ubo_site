Les difficultés avec JAVA

Écrit par Hassane SOW le 29 Novembre 2017 à 09:11
Mon amour pour JAVA n’est évident pas tout à fait cela, durant ma phase d’apprentissage, j’ai rencontré tellement de problèmes qu’à un certain moment j’ai voulu abandonner mais n’abandonnez jamais. Les problèmes que j’ai rencontrés seront rencontrés par la majorité des gens, j’en suis sûr, mais c’est peut-être parce que j’apprenais seul sans l’aide d’un professeur mais de toutes façon on va en parler quand même.

Les objets et les notions de la programmation orientée objet

Difficile de voir le rapport entre les objets et la programmation quand on commence à programmer, c’est une notion qui a été introduite dans les années « 70 »  et qui est de nos jours une notion disons obligatoire à comprendre car presque tous les langages de programmation sont orientés objets et le modèle objet rend le code plus lisible et maintenable.

La programmation asynchrone

Encore un concept qui m’a coûté beaucoup de temps avant de le comprendre, c’est la programmation asynchrone qui consiste en gros à exécuter un code en arrière plan en évitant de bloquer le thread principal, c’est une technique assez courante qui peut être utilisée par exemple lors de la lecture de gros fichier où plein d’autres actions qui nécessiteraient beaucoup de temps. La majorité des applications utilisent cette notion car en java il est conseillé d’utiliser l’edt qui est un thread pour les interfaces graphiques.

La programmation en java peut-être compliquée selon le niveau qu’on avait avant de commencer le java mais ceci n’est qu’une liste exhaustive de mes problèmes reste par exemple le problème le dur que j’ai eu lorsqu’il fallait que je comprendre le pattern observer par exemple, en plus la conception des interfaces graphiques n’a pas été facile pour moi, la compréhension du framework collections aussi et plein de choses.
